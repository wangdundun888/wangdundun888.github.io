# 包和工具
### 1.包简介
```
    为了简化设计和维护
```
### 2.导入路径
```
    import "fmt"
```
### 3.包声明
```
    package 包名//每个go文件必须有,方便其他包导入当前包的一个标识符
    tips:
        包名为main,给go build命令一个信息:编译完成后,须调用连接器生成一个可执行程序
        "_"或"."开头的源文件会被构建工具忽略
```
### 4.导入声明
```
    导入报名相同的包时,可以给其中一些包起别名
    例:
        import {
            "cypto/rand"
            mrand "math/rand"//mrand即为math/rand包的别名
        }
    起别名的优点:
        ① 解决冲突
        ② 简化长报名
        ③ 有些包名可能与变量名的意思冲突,提高程序可读性
```
### 5.包的匿名导入
```
    格式:
        import _ "fmt" //下划线做包的别名
    为什么要导入匿名包?
        因为只想要利用导入包而产生的副作用,但导入不使用会是一个编译错误
```
### 6.工具
```
    ① GOPATH对应的工作区目录
        GOPATH
            -src  //存储源代码
            -bin  //保存编译后的可执行程序
            -pkg  //保存编译后包的目标文件
    ② GOROOT
        go的安装目录,自带的标准库包的位置
    ③ go get
        下载包
```
